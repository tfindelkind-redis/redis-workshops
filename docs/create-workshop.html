What is the Workshop Builder?

A complete visual interface for creating and managing Redis workshops without touching the command line. Perfect for educators, content creators, and anyone who wants to build workshops quickly.

‚ú® Key Features:
  • Visual Module Selection - Browse and add modules with one click
  • Drag & Drop Ordering - Rearrange modules easily
  • Real-time Preview - See your workshop structure instantly
  • Navigation Preview - Visualize how learners will navigate
  • Export to JSON - Generate workshop.config.json file
  • Browser Storage - Save and load workshops locally
  • Helpful Tooltips - Guidance on every field

üìñ Detailed Guide

Planning Your Workshop

Before creating, consider these questions:

Target Audience

  • Skill Level: Beginner, Intermediate, or Advanced?
  • Prerequisites: What should participants know beforehand?
  • Goals: What will they be able to do after completion?

Workshop Structure

  • Duration: 2-3 hours (short), 4-6 hours (medium), full day (long)
  • Format: Concept-focused, project-based, or scenario-based?
  • Chapters: Which existing chapters can you reuse? What new ones do you need?
# See all available chapters to reuse ./shared/tools/list-chapters.sh

Editing README.md Frontmatter

‚ú® No JSON editing needed! Just edit simple YAML frontmatter at the top of README.md:

--- workshopId: redis-advanced-patterns title: Redis Advanced Patterns description: Master advanced Redis patterns for production duration: 6 hours difficulty: advanced chapters: shared/chapters/chapter-01-getting-started, workshops/redis-advanced-patterns/chapters/caching-strategies --- # Redis Advanced Patterns Your workshop content starts here...
‚úì Only 5 required fields! Tags, repository URL, version, and chapter metadata are auto-generated.
‚ö†Ô∏è workshopId Rules:
  • Only lowercase letters (a-z), numbers (0-9), hyphens (-)
  • NO spaces, uppercase, or special characters
  • Valid: redis-streams-2025 ‚úÖ
  • Invalid: My Workshop ‚ùå redis_2025 ‚ùå

After editing, sync the config:

./shared/tools/sync-workshop-config.sh workshops/redis-advanced-patterns
üí° What gets auto-generated:
  • Tags from your title and difficulty
  • Repository URL from workshopId
  • Version (defaults to 1.0.0)
  • Last updated date
  • Chapter order, type, and required flag

Writing Your Workshop README

The README has two parts:

1. Frontmatter (Top of file)

Simple YAML metadata between --- markers:

--- workshopId: my-workshop title: My Workshop Title description: Brief description duration: 4 hours difficulty: intermediate chapters: shared/chapters/chapter-01-getting-started, workshops/my-workshop/chapters/custom-chapter ---

2. Content (Below frontmatter)

  • Overview - What the workshop covers
  • Learning Objectives - What participants will learn
  • Prerequisites - What's needed to start
  • Setup Instructions - How to get environment ready
  • Chapter List - All chapters with descriptions and links
  • Completion Checklist - Track progress

A template is automatically created when you use create-workshop.sh

Choosing Chapter Types

Shared Chapters (Recommended)

‚úì Best for: Generic Redis concepts that multiple workshops can use
  • Located in shared/chapters/
  • Centrally maintained and versioned
  • Updates benefit all workshops
  • Automatically numbered (chapter-01, 02, etc.)

To use a shared chapter:

# 1. List available shared chapters ./shared/tools/list-chapters.sh # 2. Add to your README.md frontmatter (comma-separated) chapters: shared/chapters/chapter-01-getting-started, shared/chapters/chapter-02-data-structures # 3. Sync the config ./shared/tools/sync-workshop-config.sh workshops/my-workshop

Workshop-Specific Chapters

‚ÑπÔ∏è Best for: Unique content specific to your workshop's narrative
  • Located in workshops/your-workshop/chapters/
  • Tied to a specific workshop project
  • Can be promoted to shared later

To create a workshop-specific chapter:

# 1. Create the chapter ./shared/tools/create-chapter.sh "Building the Chat Backend" --workshop="my-workshop" # 2. Add it to your README.md frontmatter (comma-separated) # Edit workshops/my-workshop/README.md: chapters: shared/chapters/chapter-01-getting-started, workshops/my-workshop/chapters/building-the-chat-backend # 3. Sync the config ./shared/tools/sync-workshop-config.sh workshops/my-workshop
‚ö†Ô∏è Important: After creating a workshop-specific chapter, you MUST add it to the chapters: line in frontmatter and sync!

Testing Your Workshop

Before publishing, ensure everything works:

1. Sync Your Config

Make sure your workshop.config.json is up to date with your README frontmatter:

./shared/tools/sync-workshop-config.sh workshops/my-workshop

2. Validate Structure

./shared/tools/validate-workshop.sh workshops/my-workshop

3. Test Setup Script

cd workshops/my-workshop ./setup.sh

4. Walk Through Chapters

  • Complete each chapter in order
  • Verify all code examples work
  • Test exercise solutions
  • Check estimated times are realistic

5. Get Feedback

  • Have someone else try the workshop
  • Note where they get stuck
  • Improve unclear sections

Publishing Your Workshop

1. Final Validation & Sync

# Sync frontmatter to config ./shared/tools/sync-workshop-config.sh workshops/my-workshop # Validate everything ./shared/tools/validate-workshop.sh workshops/my-workshop

2. Commit to Your Branch

# On your workshop branch (created automatically) git add workshops/my-workshop git commit -m "Add My Workshop" git push -u origin workshop/my-workshop

3. Create Pull Request

üéâ Go to GitHub and create a PR:
  • Title: Add workshop: My Workshop Name
  • Complete the PR checklist
  • Wait for review and approval
  • Once merged, automatically deploys to GitHub Pages!

4. After Merge

‚ú® Automatic deployment:
  • Workshop appears on this website within minutes
  • Available in searchable catalog
  • Ready for users worldwide!
Remember: All workshops go through Pull Request review before merging to main. This ensures quality and catches issues early!

üí° Best Practices

Content Quality

  • Start with clear, achievable learning objectives
  • Provide hands-on exercises, not just theory
  • Include working code examples
  • Offer solutions to all exercises
  • Add troubleshooting sections for common issues
  • Link to official Redis documentation
  • Keep explanations clear and concise

Structure & Organization

  • Build difficulty progressively
  • Reuse existing chapters when possible
  • Keep workshops focused (2-6 hours ideal)
  • Number chapters in logical order
  • Provide estimated completion times

Maintenance

  • Test with latest Redis version
  • Update when Redis features change
  • Respond to learner feedback
  • Keep frontmatter current (edit README.md, then sync)
  • Update version number in frontmatter when making changes
  • Fix broken links or outdated references
üí° Tip: Always edit README.md frontmatter first, then run sync script. Never manually edit workshop.config.json!
